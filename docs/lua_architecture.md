# CAS 全栈纵深防御链路 — 动态策略编排引擎架构白皮书

## 一、 整体架构哲学
本项目引入了动态策略编排架构，核心理念是 "C++ 提供底层原子基础设施，Lua 脚本编排运行时动态安全策略"。通过将频繁变动的授权验证状态机、宿主环境指标审计、渲染管道流控等逻辑下沉至轻量级虚拟化脚本引擎中，使 C++ 核心层保持绝对的无状态与高内聚，从而有效切断逆向工程的静态分析流控制图谱。

## 二、 策略脚本模块权责切分
1. `core_config_offsets.lua`: 运行时环境动态特征映射表。提供多环境动态路由决策。
2. `core_auth_state.lua`: 非阻塞式身份凭证校验与多阶段认证状态机。引入两秒分帧采样网关门禁与影子写入（Shadow Write）技术，实现凭证隔离。
3. `core_main_router.lua`: 宿主进程内策略总控路由，是 C++ 引擎每 100ms 脉搏动力泵调用的唯一全局上游桩点，负责维持非阻塞的隔离区呼吸自愈流。
4. `core_policy_trigger.lua`: 进阶安全拦截钩子触发与跨进程多维排队互斥锁调度。

## 三、 C++ 侧内嵌 Lua 执行引擎设计
C++ 核心类 `AutomatedPolicyEngine` 严格控制虚拟机的生命周期。在启动时安全装载内存字节码，并通过静态符号映射向脚本环境导出 39 个高密级底层原子操作 API（涵盖内存监控、异步窗体压制、消息泵循环、跨进程互斥及物理熔断自毁等分类）。C++ 动力泵维持 100ms 一次的高频 Tick，通过无异常错误流 `ResultT<T>` 承接脚本层逻辑，并在执行完毕后对所有中间存根强制调用 `SecureZeroMemory` 进行物理级内存粉碎。
